# Digital Twin City — コンセプトアプリ要件定義

## 概要

### コンセプト

「AIデジタルツインがソフトウェアを自律的にテスト・改善していく未来」を、Cities: Skylines風のビジュアルで表現するインタラクティブなコンセプトデモ。題材として**物流・配送管理システム**を扱う。現実の物流網（倉庫・配送拠点・ルート・車両）をデジタルツインとしてソフトウェア上に再現し、AIエージェントがその仮想世界の中で人間の配送ドライバーや倉庫スタッフのように振る舞うことで、システムの問題を自律的に発見する。

### コアメッセージ

- 現実世界の物流網がデジタルツインとしてソフトウェア上に再現される
- AIエージェントがその仮想世界で人間のように振る舞い（配送・荷受け・ルート選択など）、自律的にシステムをテスト・フィードバックする
- ただしAIだけでは知り得ない現実世界の知識（道路事情・天候影響・現場の暗黙知など）は、Agent Skillsとして外部から注入される。これによりデジタルツインの「分布外」の弱点を補完する
- 完全なクローズドループではない——**人間がプロダクトビジョンを定義し、進化の方向性を決める（セミクローズドループ）**
- 生物進化のアナロジー：突然変異と自然選択はAIに、「どの生態系で戦うか」は人間が決める。Agent Skillsは「環境からの情報」——AIが自力では知覚できない現実世界のシグナルを届ける。

### 用途

個人の思考整理・ポートフォリオ用

### 技術スタック

- React（単一ページ、フロントエンドのみ）
- モックデータ使用（APIなし）
- Tailwind CSS でスタイリング
- OpenAI Images API でエージェントのスプライト画像を事前生成（ビルド時アセット）

## メタファーマッピング

このアプリでは「街（City）＝ 現実の物流ネットワークのデジタルツイン」であり、その上で動く物流管理ソフトウェアが改善対象のプロダクトとなる。

| Cities: Skylines の要素       | 本アプリでの意味                                                                                           |
| ----------------------------- | ---------------------------------------------------------------------------------------------------------- |
| 街（City）全体                | 現実の物流ネットワークのデジタルツイン                                                                     |
| 建物（Buildings）             | 物流拠点（倉庫・仕分けセンター・配送ハブ・受取ステーション等）                                             |
| 市民（Citizens）              | AIエージェント（倉庫作業員・仕分けスタッフ・配送ドライバー・受取人など、物流に関わる人間のデジタルツイン） |
| 道路・インフラ                | 配送ルート / データフロー                                                                                  |
| 街の成長・発展                | 物流システムの品質向上・配送効率の改善                                                                     |
| 市長の政策決定                | 人間によるプロダクトビジョンの注入                                                                         |
| 外部コンサルタント / 住民の声 | Agent Skills（現実世界のフィードバック・知識の注入）                                                       |
| 満足度メーター                | 配送効率スコア / 品質メトリクス                                                                            |
| 災害イベント                  | バグ・障害・予期しないエッジケース                                                                         |

## フェーズ構成

ユーザーはボタンで各フェーズを順に切り替えて体験する。各フェーズで街の見た目とダッシュボードが変化する。

### Phase 1: Deploy（初期デプロイ）

- **街の状態**: 物流拠点の建物がいくつか配置されているが、発光は最小限で薄暗い。道路はグリッド線程度。静かで寂しい印象。
- **エージェント**: 存在しない
- **ダッシュボード**: 基本メトリクスのみ（拠点数、配送成功率 —%）
- **伝えたいこと**: プロダクトの初期状態。まだフィードバックループが存在しない。

### Phase 2: Spawn Agents（エージェント投入）

- **街の状態**: Phase 1と同じ街並み
- **エージェント**: 物流に関わる人間のデジタルツインが投入される。倉庫作業員（作業服の人物スプライト、倉庫周辺で活動）、仕分けスタッフ（仕分けセンターで作業）、配送ドライバー（トラックのスプライト、道路を移動）、受取人（カジュアルな人物スプライト、受取ステーション付近で待機）など、役割ごとに異なるスプライト・行動パターンで動き始める。
- **ダッシュボード**: エージェント数が役割別に表示される。各エージェントの状態（配送中 / 荷受け中 / 待機中 / フィードバック送信中）がリアルタイムに変化。
- **伝えたいこと**: 現実世界の物流に関わる人間がデジタルツインとして再現され、プロダクトを利用し始める。

### Phase 3: Feedback Loop（フィードバック収集 + 現実世界の知識注入）

- **街の状態**: 一部の建物にマーカー（バグ発見 / 改善提案）がパルス発光で表示される。建物間を情報が流れる光のパーティクルエフェクト。さらに、街の外側（マップの端）から「現実世界のフィードバック」がシアンの光の帯として流入するエフェクト——実ユーザーの利用データや声が届く演出。
- **エージェント**: 動き回りながら、建物にフィードバックマーカー（レッドやブルーのパルス光点）を付けていく。Agent Skillsが注入されると、エージェントの行動パターンが変化する（例: 「雨天時は受け渡しに2倍かかる」というSkill注入後、配送ドライバーが受取ステーションでの滞在時間を長く取るようになる、屋根付きの受け渡しポイントを優先するようになる）。
- **ダッシュボード**: フィードバック件数がリアルタイムに増加。カテゴリ別（バグ / UX改善 / パフォーマンス）の内訳表示。Agent Skillsパネルに「外部から注入された知識」が1つずつ追加される表示（例: "現場の声: 雨天時は荷物の受け渡しに通常の2倍時間がかかる", "利用データ: 年末繁忙期に仕分けセンターで処理遅延が頻発"）。
- **伝えたいこと**: AIだけでは発見できない現実世界の知識（分布外の情報）が、Agent Skillsとしてエージェントに注入される。これによりデジタルツインの解像度が上がり、テストの質が向上する。

### Phase 4: Autonomous Evolution（自律的進化）

- **街の状態**: 建物がアップグレードされるアニメーション（スケールアップ + グロー強度が増す）。新しい建物が生えてくる。道路が拡張され光の流れが増える。ただし後半にかけて、街全体が均質化していく演出——全ての建物が同じようなサイズ・色・発光に収束し、画一的で退屈な印象になっていく。
- **エージェント**: より活発に動き回る。エージェント同士の連携を示す線が描かれる。ただし後半は似たような巡回パターンを繰り返すようになる。
- **ダッシュボード**: 品質スコアが右肩上がりのグラフ。配送成功率の上昇。修正された問題数のカウントアップ。しかし後半、スコアの上昇が鈍化・停滞する。
- **伝えたいこと**: フィードバックに基づいてプロダクトが自律的に改善されるが、方向性なき最適化は局所最適に陥る。Phase 5への布石。

### Phase 5: Vision Injection（ビジョン注入 — セミクローズドループ）

- **街の状態**: Phase 4の均質化した状態から一転。「市長（人間）」の存在がUIに現れ、ビジョンステートメントが街の上空に表示される。そこからパープルの光の波紋が同心円状に街全体に広がる演出。波紋が通過した建物は個性を取り戻し、ビジョンに沿った拠点が優先的に大きく・明るく成長する。街全体に再び生命感が戻る。
- **エージェント**: 引き続き動くが、ビジョンの方向性に沿って行動パターンが変化する。
- **ダッシュボード**: 「Human Vision」パネルが追加される。ビジョンステートメント（例: "どんな天候でも翌日届く安心を届ける"）の表示。ビジョンとの整合性スコアが新たなメトリクスとして登場。
- **伝えたいこと**: AIは「適応」が得意だが「目的の定義」はできない。人間がビジョンを注入することで、最適化の方向が定まる。完全自律ではなくセミクローズドループこそが現実的かつ強力。

## エージェントの自律的振る舞い

エージェントが「判断して行動している」ことが視覚的に伝わることが重要。スプライトによってトラックや人物として描かれたエージェントが、**意思を持った存在**として街の中で生きていること。

### 基本行動サイクル

1. **移動** — 配送ルートや拠点間を移動する（道路に沿って。移動軌跡が光のトレイルとして残る）
2. **作業** — 拠点に到着し、業務を行う（倉庫でピッキング、配送先で受け渡し等。建物のグローが強まる）
3. **判断** — 作業中にシステムの問題に遭遇するかどうか
4. **報告** — 問題を発見した場合、フィードバックマーカーを生成（パルス発光）

### 自律感を演出する振る舞い

- **エージェント間連携**: 配送トラックが配送ハブで遅延を発見 → 近くの倉庫作業員に光の線で情報を伝える → 倉庫作業員が出荷タイミングを調整しに倉庫へ向かう。「群れとしての知性」を表現する。
- **学習による行動変化**: Agent Skillsが注入される前と後で、エージェントの行動パターンが目に見えて変わる。例: 「雨天時は受け渡しに2倍かかる」というSkill注入前は全ての受取ステーションをトラックが同じように巡回 → 注入後は屋根付きの受け渡しポイントを優先し、受取ステーションでの滞在時間を長く取るようになる。
- **優先度判断**: 全ての建物を均等に回るのではなく、問題がありそうな建物に複数のエージェントが集まる。Phase 5でビジョン注入後は、ビジョンに関連する建物への訪問頻度が上がる。
- **滞在時間の差**: 問題がない建物はさっと出てくるが、複雑な問題がある建物には長く滞在する。
- **役割ごとの行動パターン**: 倉庫作業員（人物）は倉庫内で活動、仕分けスタッフ（人物）は仕分けセンターで荷物を振り分け、配送ドライバー（トラック）は道路上を走り回って拠点間を移動、受取人（人物）は受取ステーション付近で待機→荷物到着時にアクションする、など役割に応じた自然な動き。トラックと人物の視覚的な違いが、街の中の役割分担を一目で伝える。

### ステート遷移

```
idle → moving → working → [reporting | communicating] → moving → ...
```

- `working`: 拠点で業務中（建物のグローが強まり、エージェントの色が建物に染み出す）
- `reporting`: フィードバックを生成中（エージェント上にレッドまたはブルーの吹き出しがパルス発光）
- `communicating`: 他エージェントに情報を伝達中（エージェント間に光線が走り、受信側が一瞬フラッシュ）

## エージェントクリック時の詳細ビュー

マップ上のエージェント（トラックや人物）をクリックすると、そのエージェントが「実際にプロダクトを操作している様子」がポップアップまたはサイドパネルで表示される。スプライトで描かれた存在の「中身」を見せることで、デジタルツインとしてのリアリティを強化する。

### 表示内容

- **エージェントのスプライト + 名前・役割**: スプライト画像を大きく表示し、"Emily (Delivery Driver #07)" のようにラベル
- **現在の状態**: "配送ルート最適化モジュールを利用中..."
- **操作ログ（リアルタイム風）**: エージェントがプロダクトを操作しているステップが1行ずつ流れる
  ```
  > 本日の配送リストを確認（12件）
  > ルート最適化を実行
  > 最適ルートに従い配送開始
  > 3件目の配送先に到着 → 受取確認を送信
  > 5件目で不在 → 再配達登録を試行
  > ⚠️ 再配達の時間指定UIで翌日の枠が表示されない → フィードバック生成
  ```
- **発見したフィードバック一覧**: このエージェントがこれまでに報告した問題のリスト
- **保有Skills**: このエージェントに注入されているAgent Skillsの一覧

### 演出

- クリック時にマップが暗転し、対象エージェントが明るくハイライトされる（スポットライト的演出）
- 詳細パネルはダークテーマのモーダルまたはスライドインパネル。薄い輪郭グロー付き
- 操作ログは1行ずつタイプライター風に表示される（端末/コンソール風の等幅フォント、行頭に `>` のプロンプト表示）
- 閉じるボタンまたはマップクリックで元に戻る

## ヒントテキスト

あくまでデモ・説明用途のため、各フェーズで「今何が起きているか」「なぜこれが重要か」を伝えるヒントテキストを表示する。マップ上にオーバーレイで表示し、注目すべきポイントへの矢印やハイライトを伴う。

### 表示方式

- マップの上部または下部に半透明のヒントバーとして表示（ガラスモーフィズム or ダーク半透明 + ぼかし背景）
- フェーズ切替時にフェードインで登場
- 各フェーズ内で複数のヒントをステップ送り（自動 or 手動）で表示
- 注目すべき対象（エージェント、建物、パネル等）をパルスアニメーション + 点線の矢印でハイライト

### フェーズ別ヒントテキスト

**Phase 1: Deploy**

1. "これは物流管理システムのデジタルツインシティです。倉庫、仕分けセンター、配送ハブなど、現実の物流拠点が再現されています。"
2. "まだ誰もこのシステムを使っていません。品質は未知数です。"

**Phase 2: Spawn Agents**

1. "AIデジタルツインが投入されました。倉庫作業員、仕分けスタッフ、配送ドライバー、受取人——それぞれが現実の人間のように振る舞います。"
2. "エージェントは自律的に物流システムを利用し、荷物の仕分け・配送・受取を始めます。"

**Phase 3: Feedback Loop**

1. "エージェントがシステムのバグやUXの問題を発見しています。"
2. "ここで現実世界のフィードバックがAgent Skillsとして注入されます。"（街の外から流入するエフェクトをハイライト）
3. "「雨天時は荷物の受け渡しに通常の2倍かかる」——AIだけでは気づけない現実の知識が注入され、ドライバーが屋根付きポイントを優先し始めました。"（行動変化したエージェントをハイライト）

**Phase 4: Autonomous Evolution**

1. "フィードバックに基づいて、プロダクトが自動的に改善されています。"
2. "拠点がアップグレードされ、新しい配送ルートが開拓され、品質スコアが上がっていきます。"
3. "しかし——この進化には方向性がありません。全体が均質に最適化され、「何のためのプロダクトか」が見えなくなっていきます。"

**Phase 5: Vision Injection**

1. "ここで人間が介入します。プロダクトのビジョンが注入されます。"（ビジョン光の演出をハイライト）
2. "エージェントの行動がビジョンに沿って再編成されました。進化に方向性が生まれます。"
3. "AIは「適応」が得意。でも「目的」は定義できない。これがセミクローズドループ——人間とAIの理想的な役割分担です。"

### メインビュー（左側・大）

- Canvas ベースの俯瞰マップ（パフォーマンスのためCanvasを推奨）
- アイソメトリックの街並み。Cities: Skylinesのように「未来の物流都市」を感じさせるリッチな表現
- 建物は拠点の種類ごとに形状・高さ・ディテールを変える（倉庫は横長で低い、配送ハブは高層、受取ステーションは小さく点在など）
- グロー・ネオン・発光エフェクトを活用し、サイバーパンク的な未来都市感を出す
- 道路は光の線で表現し、配送ドライバーの移動軌跡が残る演出
- 背景にグリッドパターンやパーティクルを配置し、デジタル空間であることを示す
- エージェントは役割ごとのスプライト画像で表現（配送ドライバー=トラック、倉庫作業員=作業服の人物、仕分けスタッフ=人物、受取人=人物）。スムーズに移動し、発光エフェクトを伴う

### ステータスパネル（右側・小）

- ダークテーマのカードUIで、薄い境界線グローを持つ
- フェーズに応じて表示項目が増えていく
- 数値はカウントアップアニメーション付き + 変化時に微かなグロー
- Agent Skillsリストはフェーズ進行で1つずつ外部から注入されるアニメーション（シアンの光と共に出現）
- 小さな折れ線グラフ（品質スコアの推移）— グラフ線自体が発光する

### フェーズ切替ナビ（上部）

- ステップインジケーター形式。現在フェーズは発光ドット（●）、未到達は暗いドット（○）、到達済みは薄く光るドット
- 各フェーズ名をクリック/タップで切替
- 前後フェーズへの矢印ボタンもあると良い
- フェーズ間を繋ぐ線にも進行度に応じた発光

### フッター説明エリア（下部）

- 各フェーズに対応した1〜2行の説明文
- フェーズ切替時にフェードイン + タイプライター風に文字が現れる演出

## ビジュアルデザイン方針

### デザインコンセプト

「未来の物流都市」——ダークテーマをベースに、ネオンカラーの発光・グローエフェクトを多用したサイバーパンク寄りのビジュアル。デモを開いた瞬間に「すごそう」「未来だ」と感じさせることがゴール。Cities: Skylinesの夜景モードのような美しさを目指す。

### カラーパレット

- **背景**: #0a0e1a（深いダークネイビー）〜 #0f172a のグラデーション
- **建物（初期）**: #1e293b（ダークグレー）に薄い輪郭グロー
- **建物（発展後）**: #3b82f6（ブルー）や #10b981（グリーン）にネオングロー。レベルが上がるほど発光が強くなる
- **エージェント**: 役割ごとのスプライト画像 + アクセントカラーのグロー
  - 倉庫作業員: 人物スプライト + #f59e0b（アンバー）グロー
  - 仕分けスタッフ: 人物スプライト + #fb923c（オレンジ）グロー
  - 配送ドライバー: トラックスプライト + #06b6d4（シアン）グロー
  - 受取人: 人物スプライト + #a3e635（ライム）グロー
- **道路・配送ルート**: #334155 ベースに、エージェント通過時にシアンやアンバーの光が流れる
- **フィードバック**: #ef4444（レッド: バグ）/ #3b82f6（ブルー: 改善提案）— パルス発光
- **ビジョン光**: #a78bfa（パープル）— 波紋状に広がるグロー。人間の意思を象徴
- **Agent Skills流入**: #22d3ee（明るいシアン）— 街の外から流れ込む光の帯
- **テキスト**: #f8fafc（ほぼ白）、数値やラベルには微かなグロー
- **グリッド/背景パターン**: #1e293b の薄いグリッド線。デジタル空間を暗示

### エフェクト・演出

- **グロー（発光）**: 建物・エージェント・道路にCSS `box-shadow` やCanvas `shadowBlur` で発光エフェクト。フェーズ進行で発光が強まる
- **パーティクル**: 背景に漂う微小な光の粒子。データが空間を流れている印象を与える
- **軌跡（トレイル）**: 配送ドライバーの移動経路に薄い光の残像が残る
- **パルス**: フィードバック発見時、建物から放射状にパルス波が広がる
- **波紋**: Phase 5のビジョン注入時、中心から同心円状の光の波紋が街全体に広がる
- **フェーズ遷移**: ホワイトフラッシュやズームアウト等、インパクトある切替演出

### アニメーション

- エージェントの移動: requestAnimationFrame ベース。移動中に軌跡エフェクトを伴う
- 建物の成長: スケールアップ + 色変化 + グロー強度の増加を組み合わせたリッチなトランジション
- カウントアップ: 数値のイージングアニメーション + 数値変化時の微かなフラッシュ
- フェーズ切替: 画面全体に影響する演出（暗転→展開、パーティクル爆発など）
- エージェント間通信: 光線が走り、受信側エージェントが一瞬明るく光る

### レスポンシブ

- デスクトップファースト（ポートフォリオ閲覧を想定）
- 最低限タブレットでも閲覧可能

## スプライトアセット

エージェントを「人」や「トラック」として表現するため、OpenAI Images APIでアイソメトリック視点のスプライト画像を事前生成し、プロジェクトにアセットとして同梱する。ランタイムでのAPI呼び出しは行わない。

### 生成するスプライト一覧

| 役割           | 見た目                                             | バリエーション  |
| -------------- | -------------------------------------------------- | --------------- |
| 配送ドライバー | 小型配送トラック（シアンのアクセントカラー）       | 移動中 / 停車中 |
| 倉庫作業員     | 作業服を着た人物（アンバーのアクセントカラー）     | 歩行中 / 作業中 |
| 仕分けスタッフ | 作業服を着た人物（オレンジのアクセントカラー）     | 歩行中 / 作業中 |
| 受取人         | カジュアルな服装の人物（ライムのアクセントカラー） | 待機中 / 受取中 |

### 生成方針

- 画風: アイソメトリック視点、ダークテーマの背景に映えるように明るめの色調。ミニチュア/ローポリ風のスタイライズされたデザイン
- サイズ: 小さめ（マップ上で32〜64px程度で表示されるため、シンプルで視認性の高いデザイン）
- 背景: 透過PNG
- 統一感: 全スプライトが同一のアートスタイルで統一されていること

### 建物アセット（任意）

建物もスプライトで生成すると統一感が出る。余力があれば以下も生成する：

| 拠点             | 見た目                            |
| ---------------- | --------------------------------- |
| 中央倉庫         | 大型の横長倉庫。搬入口あり        |
| 仕分けセンター   | 中型のベルトコンベア付き施設      |
| 配送ハブ         | トラックの発着場がある中継拠点    |
| 受取ステーション | 小型のロッカー/カウンター付き施設 |

建物スプライトを生成しない場合は、Canvasでアイソメトリックな立体図形として描画する（その場合も影・グラデーション・グローで立体感を出すこと）。

## フェーズ別フッター説明文

| Phase                    | 説明文                                                                                                     |
| ------------------------ | ---------------------------------------------------------------------------------------------------------- |
| 1 - Deploy               | プロダクトがデプロイされた。まだフィードバックループは存在しない。                                         |
| 2 - Spawn Agents         | AIデジタルツインが投入され、プロダクトの自律的な探索が始まる。                                             |
| 3 - Feedback Loop        | エージェントが問題を発見する。同時に、AIだけでは知り得ない現実世界の知識がAgent Skillsとして注入される。   |
| 4 - Autonomous Evolution | フィードバックに基づき、プロダクトが自律的に進化する。しかし方向性なき最適化は、やがて停滞する。           |
| 5 - Vision Injection     | しかしAIは「目的」を定義できない。人間がビジョンを注入し、進化の方向を決める。これがセミクローズドループ。 |

## 実装上の補足

### 実装方針

本ドキュメントはコンセプトと演出の意図を定義するものであり、実装の詳細（マップサイズ・座標系・建物の数と配置・エージェント数・道路構造・フェーズ遷移のタイミング・ヒントテキストの送り方式など）は開発者の判断に委ねる。判断基準は以下の2点：

> 1. **「このデモを見た人が、AIによるソフトウェアの自律的進化とセミクローズドループの概念を直感的に理解できるか」**
> 2. **「デモを開いた瞬間に『すごい、これが未来だ』と感じさせるビジュアルのインパクトがあるか」**

コンセプトの伝達力とビジュアルの没入感の両方を重視する。チープな見た目ではコンセプトの説得力も落ちる。

### 優先度

1. **最優先**: フェーズ切替 + リッチなメインマップ描画（グロー・発光・立体感のあるアイソメトリック街並み）+ スプライトベースのエージェント（トラック・人物）の自律的移動アニメーション（軌跡エフェクト付き）
2. **高**: ステータスパネルのメトリクス表示 + フェーズ連動 + ヒントテキスト表示
3. **中**: エージェント間連携の光線演出 + 建物のアップグレードアニメーション + Agent Skills注入による行動変化 + パーティクルエフェクト
4. **低**: Phase 4の均質化演出 + レスポンシブ対応

### 技術的な推奨

- マップ描画: HTML5 Canvas（グロー・パーティクル・軌跡エフェクトのパフォーマンスのため必須級）
- スプライト: OpenAI Images APIで事前生成した透過PNGをCanvas `drawImage` で描画。エージェントの発光はスプライトの周囲に `shadowBlur` で表現
- アイソメトリック表現: 2Dで斜め45度風の投影。3Dライブラリは不要だが、影やグラデーションで立体感を演出する
- エフェクト: Canvas `shadowBlur` でグロー、`globalAlpha` で軌跡の減衰、`globalCompositeOperation: 'lighter'` で加算合成による発光表現
- アニメーション: requestAnimationFrame ベースの描画ループ。60fps目標
- 状態管理: useState / useReducer で十分（外部ライブラリ不要）

### やらないこと

- バックエンドAPI連携
- ランタイムでのOpenAI Images API呼び出し（スプライトは事前生成してアセットとして同梱）
- ユーザー認証
- データ永続化
- モバイル最適化（閲覧可能レベルで十分）
