# Digital Twin City — コンセプトアプリ要件定義

## 概要

### コンセプト

「AIデジタルツインがソフトウェアを自律的にテスト・改善していく未来」を、ペーパーマリオ/ヨッシーアイランド風のビジュアルで表現するインタラクティブなコンセプトデモ。題材として**物流・配送管理システム**を扱う。現実の物流網（倉庫・配送拠点・ルート・車両）をデジタルツインとしてソフトウェア上に再現し、AIエージェントがその仮想世界の中で人間の配送ドライバーや倉庫スタッフのように振る舞うことで、システムの問題を自律的に発見する。

### コアメッセージ

- 現実世界の物流網がデジタルツインとしてソフトウェア上に再現される
- AIエージェントがその仮想世界で人間のように振る舞い（配送・荷受け・ルート選択など）、自律的にシステムをテスト・フィードバックする
- ただしAIだけでは知り得ない現実世界の知識（道路事情・天候影響・現場の暗黙知など）は、**外部ナレッジ**として注入される。これはエージェント個人の属性ではなく、プロダクト改善の判断基準として外部から補完される知識である
- 完全なクローズドループではない——**人間がプロダクトビジョンを定義し、進化の方向性を決める（セミクローズドループ）**
- 生物進化のアナロジー：突然変異と自然選択はAIに、「どの生態系で戦うか」は人間が決める。外部ナレッジは「環境からの情報」——AIが自力では知覚できない現実世界のシグナルを届ける。

### 用途

個人の思考整理・ポートフォリオ用

---

## メタファーマッピング

このアプリでは「街（City）＝ 現実の物流ネットワークのデジタルツイン」であり、その上で動く物流管理ソフトウェアが改善対象のプロダクトとなる。

| Cities: Skylines の要素       | 本アプリでの意味                                                                                           |
| ----------------------------- | ---------------------------------------------------------------------------------------------------------- |
| 街（City）全体                | 現実の物流ネットワークのデジタルツイン                                                                     |
| 建物（Buildings）             | 物流拠点（倉庫・仕分けセンター・配送ハブ・受取ステーション等）                                             |
| 市民（Citizens）              | AIエージェント（倉庫作業員・仕分けスタッフ・配送ドライバー・受取人など、物流に関わる人間のデジタルツイン） |
| 道路・インフラ                | 配送ルート / データフロー                                                                                  |
| 街の成長・発展                | 物流システムの品質向上・配送効率の改善                                                                     |
| 市長の政策決定                | 人間によるプロダクトビジョンの注入                                                                         |
| 外部コンサルタント / 住民の声 | 外部ナレッジ（現実世界のフィードバック・知識の注入）                                                       |
| 満足度メーター                | 配送効率スコア / 品質メトリクス                                                                            |
| 災害イベント                  | バグ・障害・予期しないエッジケース                                                                         |

## フェーズ構成

ユーザーはヘッダー上のナビゲーションで各フェーズを順に切り替えて体験する。各フェーズで街の見た目とダッシュボードが変化する。

### Phase 1: Deploy（初期デプロイ）

- **街の状態**: 物流拠点の建物がいくつか配置されている。静かで寂しい印象。
- **エージェント**: 存在しない
- **ダッシュボード**: 基本メトリクスのみ（拠点数、配送成功率 —%）
- **伝えたいこと**: プロダクトの初期状態。まだフィードバックループが存在しない。

### Phase 2: Spawn Agents（エージェント投入）

- **街の状態**: Phase 1と同じ街並み
- **エージェント**: 物流に関わる人間のデジタルツインが投入される。倉庫作業員、仕分けスタッフ、配送ドライバー、受取人など、役割ごとに異なるスプライト・行動パターンで動き始める。
- **ダッシュボード**: エージェント数が役割別に表示される。各エージェントの状態がリアルタイムに変化。
- **伝えたいこと**: 現実世界の物流に関わる人間がデジタルツインとして再現され、プロダクトを利用し始める。

### Phase 3: Feedback Loop（フィードバック収集 + 外部ナレッジ注入）

- **街の状態**: 一部の建物にマーカー（バグ発見 / 改善提案）が表示される。
- **エージェント**: 動き回りながら、建物にフィードバックマーカーを付けていく。
- **ダッシュボード**: フィードバック件数がリアルタイムに増加。カテゴリ別（バグ / UX改善 / パフォーマンス）の内訳表示。外部ナレッジパネルに「外部から注入された知識」が表示される。
- **伝えたいこと**: AIだけでは発見できない現実世界の知識（分布外の情報）が、外部ナレッジとして注入される。これによりデジタルツインの解像度が上がり、テストの質が向上する。

### Phase 4: Autonomous Evolution（自律的進化）

- **街の状態**: 建物がアップグレードされるアニメーション。
- **エージェント**: より活発に動き回る。
- **ダッシュボード**: 品質スコアが右肩上がりのグラフ。配送成功率の上昇。修正された問題数のカウントアップ。
- **伝えたいこと**: フィードバックに基づいてプロダクトが自律的に改善されるが、方向性なき最適化は局所最適に陥る。Phase 5への布石。

### Phase 5: Vision Injection（ビジョン注入 — セミクローズドループ）

- **街の状態**: ビジョン注入による演出。
- **エージェント**: ビジョンの方向性に沿って行動パターンが変化する。
- **ダッシュボード**: ビジョンとの整合性スコアが新たなメトリクスとして登場。
- **伝えたいこと**: AIは「適応」が得意だが「目的の定義」はできない。人間がビジョンを注入することで、最適化の方向が定まる。完全自律ではなくセミクローズドループこそが現実的かつ強力。

## 外部ナレッジの概念

従来「Agent Skills」と呼んでいた概念を「**外部ナレッジ**」に改名。

### 定義

AIだけでは知り得ない現実世界の知識。プロダクト改善の判断基準として外部から注入される。

### 重要な区別

- 外部ナレッジは**エージェント個人の属性ではない**
- プロダクト全体の改善に寄与する**外部知識ソース**である
- ソースの種類: 現場フィードバック、利用データ、専門家知見

### 例

- 「雨天時は荷物の受け渡しに通常の2倍時間がかかる」（現場FB）
- 「年末繁忙期に仕分けセンターで処理遅延が頻発」（利用データ）
- 「冷蔵品は30分以内に届けないと品質劣化」（専門家）

## セミクローズドループ

本アプリの中心概念。ヒューマンビジョンモーダルで視覚的に説明される。

### フロー

1. **AI探索** — エージェントがプロダクトを自律的に利用・テスト
2. **フィードバック発見** — バグ・UX問題・パフォーマンス課題を検出
3. **外部ナレッジ注入** — AIだけでは知り得ない現実世界の知識を補完
4. **プロダクト改善** — フィードバックに基づく自律的な品質向上
5. **人間のビジョン** — プロダクトの目的と方向性を人間が定義

→ ビジョンに沿って再探索（ループ継続）

### 核心メッセージ

> AIは「適応」が得意。でも「目的」は定義できない。
> これがセミクローズドループ——人間とAIの理想的な役割分担。

## ヒューマンビジョン

ヒューマンビジョンは**全フェーズから独立してアクセス可能**なモーダルとして実装。マップ左下のボタンから開く。

### 内容

- ビジョンステートメント（例: "どんな天候でも翌日届く安心を届ける"）
- 優先事項リスト
- セミクローズドループのタイムライン図（フェーズ進行に応じて段階的に活性化）

---

## エージェントの自律的振る舞い

エージェントが「判断して行動している」ことが視覚的に伝わることが重要。スプライトによってトラックや人物として描かれたエージェントが、**意思を持った存在**として街の中で生きていること。

### 基本行動サイクル

1. **移動** — 配送ルートや拠点間を移動する
2. **作業** — 拠点に到着し、業務を行う
3. **判断** — 作業中にシステムの問題に遭遇するかどうか
4. **報告** — 問題を発見した場合、フィードバックマーカーを生成

### ステート遷移

```
idle → moving → working → [reporting | communicating] → moving → ...
```

---

## エージェントクリック時の詳細ビュー

マップ上のエージェントをクリックすると、そのエージェントが「実際にプロダクトを操作している様子」がスライドインパネルで表示される。

### 表示内容

- **エージェントのスプライト + 名前・役割**
- **現在の状態**
- **操作ログ（リアルタイム風）**: タイプライター演出で1行ずつ表示
- **発見したフィードバック一覧**

---

## ヒントテキスト

各フェーズで「今何が起きているか」「なぜこれが重要か」を伝えるヒントテキストをマップ上部にオーバーレイ表示する。フェーズ切替時にフェードインで登場し、ステップ送りで複数ヒントを表示可能。

## ビジュアルデザイン方針

### デザインコンセプト

「ペーパーマリオ / ヨッシーアイランド風の手作り感のある温かいビジュアル」——暖色パステル系のカラーパレットで、クラフトペーパーのような質感を持つUI。

### レスポンシブ

- デスクトップファースト（ポートフォリオ閲覧を想定）
- 最低限タブレットでも閲覧可能
